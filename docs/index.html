pg_dbml

Extraktor schématu databáze

Exportujte celé schéma PostgreSQL do formátu DBML jediným příkazem. Čisté SQL, žádné externí závislosti.

Zobrazit na GitHubu Rychlý start ↓
terminal

# Export schématu do DBML

$ ./pg_dbml \

--db-host localhost \

--db-name my_database \

--output schema.dbml

✓ Schéma úspěšně exportováno → schema.dbml▌

Co je pg_dbml?

pg_dbml je výkonný nástroj příkazového řádku navržený k introspekci schématu databáze PostgreSQL a exportu jeho kompletní struktury do formátu Database Markup Language (DBML). Toho dosahuje prováděním komplexních čistých SQL dotazů na systémové katalogy PostgreSQL, čímž zajišťuje 100% věrnost schématu bez nutnosti externích ovladačů nebo ORM.

💡 Proč používat DBML pro AI a LLM?

Databázová schémata jsou složitá na zpracování. Přímé dotazování živé databáze může být pomalé, drahé a vyžaduje hlubokou znalost SQL. DBML tento problém řeší tím, že poskytuje čitelný, deklarativní a vysoce strukturovaný plán vašich dat.

✦ Přínos pro AI

Konzumací souboru DBML mohou velké jazykové modely (LLM) rychle a nákladově efektivně porozumět vašemu datovému modelu. Okamžitě pochopí vztahy, struktury tabulek a omezení, aniž by musely spouštět časově náročné dotazy na produkční databázi.

Klíčové funkce

⬡

Čistá SQL introspekce

Logika je zcela obsažena v PostgreSQL dotazech — žádné externí závislosti.

⛓

Mapování omezení

Detekuje primární klíče, unikátní omezení a vztahy cizích klíčů.

◈

Přehled indexů

Dokumentuje vlastní indexy a sloupce INCLUDE pro úplné pokrytí schématu.

◉

Bohatost metadat

Zahrnuje komentáře tabulek a sloupců, pokud jsou k dispozici.

▶

Řízeno CLI

Jednoduché, opakovatelné spouštění přes příkaz pg_dbml. Vhodné pro skripty a CI.

Pro systémy bez Homebrew nebo pokud chcete mít kód pod kontrolou.

2

Povolení spuštění

chmod +x pg_dbml
3

Pro globální dostupnost:

ln -s $(pwd)/pg_dbml /usr/local/bin/pg_dbml

* Vyžaduje nainstalovaný psql v PATH.

Použití a parametry

Nástroj pg_dbml můžete spouštět buď s jednotlivými parametry, nebo pomocí standardního PostgreSQL Connection URI.

Základní spuštění

terminal

# Export podle názvu databáze (host: localhost)

$ pg_dbml -d my_database -o schema.dbml

Použití Connection URI

terminal

# Export pomocí úplné adresy

$ pg_dbml postgresql://user:pass@localhost:5432/my_db

Náhled (dry-run)

terminal

# Náhled bez zápisu do souboru

$ pg_dbml -d mydb --dry-run

Přehled parametrů

Parametr Zkratka Popis Výchozí
--dbname -d Název cílové databáze. -
--host -h Adresa PostgreSQL serveru. localhost
--port -p Port serveru. 5432
--user -U Uživatelské jméno. postgres
--output -o Cesta k výslednému souboru. [DBNAME].dbml
--quiet -q Potlačí výpis úspěchu. -
--dry-run - Náhled bez zápisu. -
--version -v Zobrazí verzi skriptu. -

Technická architektura

Projekt je rozdělen do dvou klíčových částí pro maximální udržovatelnost a skvělý vývojářský zážitek:

pg_dbml (Bash orchestrátor)

Zajišťuje parsování argumentů, bezpečné připojení k databázi přes psql a zápis streamu do souboru.

pg_dbml.sql (Jádro)

Komplexní, čistě SQL dotaz nad systémovými tabulkami PostgreSQL. Díky oddělení v samostatném souboru těžíte z plného zvýraznění syntaxe a standardního SQL ladění.
